The Grid

The grid is essential for a website to be responsive - adapting properly for all devices, from phone to desktop to jumbotron! The grid can support a maximum 12 columns. But don't worry, you can have any number of columns, as long as they don't go over 12!

Grid options

You define column size by using one of 4 dynamic css classes. Each class is designed to adjust column sizes relative to different device sizes

Extra small devices Phones (<768px) Small devices Tablets (≥768px) Medium devices Desktops (≥992px) Large devices Desktops (≥1200px)
Grid behavior Horizontal at all times Collapsed to start, horizontal above breakpoints
Container width None (auto) 750px 970px 1170px
Class prefix .col-xs- .col-sm- .col-md- .col-lg-
# of columns 12
Column width Auto ~62px ~81px ~97px
Gutter width 30px (15px on each side of a column)
Nestable Yes
Offsets Yes
Column ordering Yes

Grid creation

To create a grid you must define rows and then put columns inside those rows.

Page Slices

Use the .page-slice class to distinguish separate content sections of a page.

Note that .page-slice is built into the page with menuslayout component

I'm a page slice

Breaking up content into slices like this really makes it easier to skim!

   <p>...</p>
 </div>

Striped Page Slices

Sometimes pages have many content sections. To help delineate before pages with many content sections you can use the .page-slice-striped class along side the .page-slice to create page slices with alternating background colors.

Page Slice

Page slices with alternating background color can help delineate pages with a lot of page slices

I'm striped!

.....

                             <div class="page-slice page-slice-striped">
   <h2>Page slice title</h2>
   <p>...</p>
 </div>

Pages & Submenus

Submenus are automatically generated by useing the Page component. This allows content creators to focus on content without having to worry about navigation. Assign the .page-with-menu class to the container that will hold each "menued" section of the page, then add the .page-section class to each section.

Menu items are assigned using the <h2> heading or the title attribute for each .page-section. Always use the title attribute for long section headings.

You can also assign menu icons for each section using the icon attribute for each .page-section.

We may need to removed the icon feature from the docs, as it will likely only be used in special cases
